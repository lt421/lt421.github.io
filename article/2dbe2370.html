c语言简介、基本数据类型

一、C语言发展

与UNIX 密不可分，汇编语言编写的UNIX，UNIX使用c语言改写，从此编写操作系统的主要语言

至今仍是排名靠前语言

主流版本： c89(标准c)，c99

c语言特点：

  • 代码级别的跨平台
  • 允许直接访问物理地址，对硬件进行操作：（操作系统，数据库，杀毒软件，C语言，驱动…）
  • 传值方式：按值/按指针
  • 没有对象 - 结构体

二、C程序运机制

  1. 编辑：编写.c文件，就是源代码
  2. 编译：将.c文件翻译成目标文件(.obj) //计算机底层
  3. 链接：将目标文件.obj + 库文件 生成可执行文件(.exe) //计算机底层
  4. 运行：执行.exe文件，得到运行结果

编译：

有了c源文件(xxx.c)，通过编译器将其编译成obj文件(目标文件)。
如果程序没有错误，没有任何提示，但在Debug目录下会出现一个（程序名）.obj文件，该文件称为目标文件。

链接：

有了目标文件（.obj文件）,通过链接程序将其和运行所需要的C库文件链接成exe文件（可执行文件）。
如果程序没有错误，没有任何提示，但在Debug目录下会出现一个项目名.exe文件，该文件称为可执行文件。

为什么需要链接库文件？ 使用printf等函数是c程序库中提供，并不是自己写的（一个printf函数有上千行代码）。
链接后的.exe比.obj大很多（还包含了库文件）。

运行：

链接后生成的.exe文件（二进制文件）。
可以在控制台直接运行。

区别：

  1. const带类型，define不带类型

  2. const在编译，运行时才起作用，define在编译的预处理阶段起作用

  3. define只是简单的替换，没有类型检查，简单的字符串替换会导致边界效应

  4. const可以进行调试，而define不行，预编译阶段以及替换，调试的时候就没有了

  5. const不能重复定义，而define可以通过undef取消某个符号的定义，再重新定义

  6. define还可以配合#ifdef,#ifndef,#endif 来使用，使代码更加灵活。比如通过#define来启动或关闭调试信息

4.1、static关键字

static关键字在c语言中比较常用，使用恰当能够大大提高程序的模块化特性，有利于拓展和维护

局部变量使用static修饰：

  1. 称为静态局部变量
  2. 自动初始化
  3. 存储于内存的静态存储区，只会初始一次，即使函数返回，它的值也保持不变

全局变量使用static修饰：

  1. 普通全局变量对整个工程可见，其他文件可以使用extern外部声明后直接使用，也就是其他文件不能再定义一个相同名字的变量了（否则编译器会认为他们是一个变量），静态全局变量仅对当前文件可见，其他文件不可访问。
    报错：
    CMakeFiles\day01.dir/objects.a(static.c.obj):G:/clion_project/basic/day01/static.c:9: multiple definition of `n’
    CMakeFiles\day01.dir/objects.a(function.c.obj):G:/clion_project/basic/day01/head/function.c:39: first defined here
    若要使用：
    extern int n;
  2. 定义不需要与其他文件共享的全局变量时，加上static关键字能够有效的降低程序之间的耦合，避免不同变量的冲突，且不会误用

函数使用static:

  1. 非静态函数可以在另一个文件中通过extern引用
  2. 静态函数只能在声明它的文件中可见，其他文件不能引用该函数

2.2、宏定义

#define叫做宏定义命令，用一个标识符标识一个字符串，源文件出现标识符进行简单的替换

宏定义的形式：

  1. 宏名命名与变量相同；字符串可以是数字，表法式，if语句，函数等
  2. 程序中反复使用的表达式就可以使用宏定义

Notes

  1. 预处理程序对表达式不作任何检查，若有错误，只能在编译已被宏展开的的源程序时发现
  2. 宏定义不需要加；（若加了会把;也一起替换）
  3. 作用域为源程序结束，终止可以使用#undef命令
  4. 习惯使用大写（也可以使用小写）
  5. 可以使用宏定义表示数据类型，书写方便
    #define UINT unsigned int
  6. 与typedef区别：宏定义只是简单的字符串代替，而typedef是在编译阶段由编译器处理的，给原有的数据类型起新的名字，作为一种新的数据类型

2.3、带参数的宏定义

形式参数之间可以出现空格，但是宏名与形式参数之间不能有空格

2.4、C语言预处理命令总结

指令 说明
# 空指令，没有任何效果
#include 引入头文件
#define 宏定义
#undef 取消宏定义
#if 如果条件为真，则编译
#ifdef 如果宏已经定义，则编译

#elif else if 为真则编译

一、动态内存分配

1.1、内存布局

c语言中，不同数据在内存中分配的说明

  1. 全局变量 – 内存中的静态存储区
  2. 非全局局部变量 – 内存中的动态存储区 – stack栈
  3. 临时使用的数据 – 建立动态内存分配区域，需要随时开辟，不需要时及时释放– heap堆
  4. 根据需要向系统申请所需大小的空间，由于未在声明部分定义其为变量或数组，不能通过变量或数组名来引用这些数据，只能通过指针来（引用）

1.2、内存动态分配相关函数

头文件 #include <stdlib.h>

返回值类型：

c99标准，返回类型void*为无类型指针，即不指向哪一种具体的数据类型，只表示一个纯地址，不指向任何具体的对象

当把void指针赋值给不同类型的指针变量（或相反）时，编译器会自动进行转换

Notes:

  1. 避免分配大量小的内存块，分配堆上的内存有系统开销，分配多个小的内存要比分配几个大的内存块开销大
  2. 仅在需要时分配内存，使用完及时释放，否则可能出现内存泄露
  3. 在释放内存前，确保不会无意中覆盖堆上已经分配的内存地址，否则程序就会出现内存泄露。在循环分配内存时，要特别小心

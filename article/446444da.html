一、动态内存分配

1.1、内存布局

c语言中，不同数据在内存中分配的说明

  1. 全局变量 – 内存中的静态存储区
  2. 非全局局部变量 – 内存中的动态存储区 – stack栈
  3. 临时使用的数据 – 建立动态内存分配区域，需要随时开辟，不需要时及时释放– heap堆
  4. 根据需要向系统申请所需大小的空间，由于未在声明部分定义其为变量或数组，不能通过变量或数组名来引用这些数据，只能通过指针来（引用）

1.2、内存动态分配相关函数

头文件 #include <stdlib.h>

返回值类型：

c99标准，返回类型void*为无类型指针，即不指向哪一种具体的数据类型，只表示一个纯地址，不指向任何具体的对象

当把void指针赋值给不同类型的指针变量（或相反）时，编译器会自动进行转换

Notes:

  1. 避免分配大量小的内存块，分配堆上的内存有系统开销，分配多个小的内存要比分配几个大的内存块开销大
  2. 仅在需要时分配内存，使用完及时释放，否则可能出现内存泄露
  3. 在释放内存前，确保不会无意中覆盖堆上已经分配的内存地址，否则程序就会出现内存泄露。在循环分配内存时，要特别小心

1.3、指针使用一览

变量定义 类型表示 含义
int i int 定义整型变量
int *p int * 定义整型指针变量
int a[5] int[5] 定义整数数组
int *p[4] int*[4] 指针数组，它由四个指向整型数据的指针组成
int (*p)[4] int(*)[4] 数组指针，p为指向包含4个元素的一维数组的指针变量
int f() int() 函数
int *p() int *() 指针函数 （返回值为指针的函数）
int (*p)() int(*)() 函数指针 （指向函数地址的指针）
int **p int** 二级指针
void *p void* 无类型指针（c99以上可以自动转换）

三、共用体

1）共用体（Union）（也称为联合或联合体）属于构造类型，它可以包含多个类型不同的成员。和结构体非常类似，但是也有不同的地方

2）结构体和共用体的区别在于：结构体的各个成员会占用不同的内存，互相之间没有影响；而共用体的所有成员占用同一段内存，修改一个成员变量会影响所有成员

一、数据库

1、事务的四大特性

ACDI；
原子性：事务只能全部执行或者全部回滚。
一致性：事务执行前后数据库处于一致性的状态。
持久性：事务对数据库的操作对数据库数据的改变是永久性的。
隔离性：多个事务并发对数据库操作是隔离开的。

2、事务隔离级别与脏读，幻读，不可重复读等

脏读：事务A读取到未提交的数据（回滚的数据）
不可重复读：事务A读取两次数据，结果集不一样（事务B进行了数据修改）
幻读：事务A读取两次数据，结果集的数目不一样（如多或少了几条数据），事务B对数据进行了添加和删除

事务隔离级别（级别越高，并发性越低）：

read uncommit：读未提交
read commit：读已提交
repeatable read：可重复读
seizaltion：串行

Mysql默认级别：repeatable read,且使用一种xxxx技术解决幻读。

若开发需要高并发，将隔离级别设置为read uncommit，设置乐观锁/悲观锁来解决。（追问就g）

3、数据库范式

1NF：得是二维表（每一个表项都是不可分割的数据）
2NF：消除部分依赖（追问g）
3NF：消除传递依赖（追问g，那些什么候选键乱七八糟的看一次忘一次）

4、索引

索引可以加快查询的效率，但是降低其他操作的效率，因为每次插入删除或修改后需重新组织索引；

5、Nosql

key-value形式存储，如redis，特点为高性能，快。缺点为不能进行复杂查询。用于缓存热点数据或进行高并发开发。

6、视图、触发器、存储过程、游标

视图：把经常连接查询的表作为视图存储起来，方便使用。

触发器：满足某种条件时候执行触发器。

存储过程：写好具有特点功能的sql语句供上层函数调用。

游标：分别对每行元素处理的时候使用游标。

二、操作系统

1、操作系统四个特点

  • 并发：操作系统中各个进程并发执行。
  • 共享：操作系统并发执行的各个进程共享系统资源。
  • 虚拟：物理实体映射为逻辑实体。
  • 异步：各个进程以不可预知的速度，走走停停的推进。

2、操作系统主要功能

  • 作为系统资源的管理者

    • 处理机管理
    • 存储器管理
    • 文件管理
    • 设备管理

  • 作为向上提供接口

    • GUI(图形界面)
    • 系统调用
    • 命令接口（联机/脱机）

3、进程与线程的区别

进程：运行着的程序，由程序段，数据端，PCB三部分组成。

线程：轻量级的进程，进程中包含线程。

区别：
进程是资源分配的最小单位，线程是处理机调度的最小单位。
进程切换开销大，同一进程里的线程切换系统开销小。

4、进程的状态

五状态模型：创建态，就绪态，运行态，阻塞态，终止态
七状态模型：内存空间不够需要换到外存，挂起态（阻塞挂起，就绪挂起）

5、进程通信

  • 管道通信
  • 共享存储
  • 消息传递

6、处理机调度的层次

高级调度（作业调度）：作业中选择调入内存的作业
中级调度（内存调度）：将哪个挂起态的进程调入内存
低级调度（进程调度）：运行哪个就绪态的进程

7、调度算法

先来先服务
短作业优先
高响应比优先

时间片轮转
优先级
多级反馈队列

8、死锁

死锁：各进程互相等待对方手里的资源，导致各进程都阻塞，无法向前推进的现象

原因：信号量使用不当、进程推进的顺序不对。

必要条件：

  1. 互斥条件：请求互斥资源。
  2. 请求和保持条件：拿到资源还可以请求资源。
  3. 非剥夺条件：已经分配的资源不能被剥夺
  4. 循环等待条件：系统中若干进程组成环路，该环路中每个进程都在等待相邻进程正占用的资源

解决死锁：

  1. 预防死锁：破坏必要条件
  2. 避免死锁：银行家算法
  3. 检测，解决死锁不会

9、页面置换算法

1、最佳置换算法：每次选择淘汰的页面将是以后不再使用，或者在最长时间内不再被访问的页面（不知道后面要访问那个，理想状态）。

2、先进先出置换算法：每次淘汰的页面是最早进入内存的界面 。

3、最近最久未使用算法：每次淘汰的页面是最近最久未使用的页面。标志位T记录使用的时间

4、时钟置换算法（CLOCK）：（细问就g）

10、虚拟内存

结果：用户使用到的内存比实际内存大的多；

基于局部性原理，在程序装入时，可以将程序中很快会用到的部分装入内存，暂时用不到的部分留在外存，就可以让程序开始执行。允许一个作业分多次调入内存。

11、分页与分段

页是信息的物理单位。分页的目的是为了实现离散分配，提高内存利用率。分页仅仅是系统管理上的需要，完全是系统行为，对用户是不可见的。

段是信息的逻辑单位。分段的目的是为了更好的满足用户的需求。一个段通常包含着一组逻辑模块的信息。分段对用户是可见的，用户编程时需要显式的给出段名

12、磁盘调度算法

1、先来先服务算法

2、最短寻找时间优先，找离磁盘头最近的

3、saca 算法：磁头移动到最外侧磁道的时候才能往内移动，移动到最内侧磁道的时候才能往外移动（电梯）

4、LOOK 调度算法：如果在磁 头移动方向上已经没有别的请求，就可以立即改变磁头移动方向

三、计算机网络

1、计算机网络的分层各层常见协议

ois:物联网淑慧适用（物理层，数据链路层，网络层，传输层，会话层，表示层，应用层）。
tcp/ip:网络接口层，网络层，传输层、应用层。
学习计网：物理层 数据链路层 网络层 传输层 应用层。

常见协议：
应用层：http ftp ssh dns fmtp..
传输层：tcp,ip
网络层：ip
数据链路层：ARP

2、tcp三次握手与四次挥手

根本原因：在不可靠的通信通道上建立可靠的连接

握手：
client 发送 syn 请求到server,server 回复 syn ack,client再发送ack请求。

挥手：
client 发送 fin 请求到server，此时client不再发送数据，sever收到请求回复ack,此时server仍可以发送和处理数据，它处理完这些数据再给client发送fin请求，客户端收到后回复ack，server收到ack后立即断开连接，client经过2msl后断开连接

为什么不是两次握手
防止已失效的报文到达服务器端（比如是两次连接，client发送第一个syn请求由于某原因没有到达server，重新发送，server收到立马建立连接，过了会失效的那个到了，就又建立起一个连接，这样在client端是一个连接，在server端是两个连接就出错）

为什么要等2msl

确保最后一个ack server端收到，否则server端一直处于等待状态

为什么四次挥手

sever收到fin报文时还需要发送报文或者处理数据，不能立即断开连接

3、TCP、UDP区别（何时使用TCP,何时使用UDP）

  1. TCP面向连接，UDP无连接。
  2. TCP可靠传输，不会发生顺序混乱，丢失这些情况。UDP不可靠
  3. TCP一对一传输，UDPp可以一对一，一对多…
  4. TCP传输基本单位是字节，UDP是数据报
  5. UDP具有实时性和高效，如直播，视频通话等使用UDP

4、浏览器输入 www.baidu.com 发生了什么

  1. DNS解析，首先会查本地的缓存有没有存储域名对应的ip地址，若没有则向DNS服务器发送请求获取域名对应的ip地址
  2. 得到ip地址后，三次握手建立tcp连接（https基于tcp）
  3. get请求，http报文 -> 加首部 tcp报文 -> 加首部 ip报文 -> 加首部尾部称为数据帧 -> 比特流转为电信号 传输。
  4. 结果路由器跳转，不断修改帧的Mac地址到目标主机又拆数据报，然后相同的方式响应请求。

5、TCP流量控制与拥塞控制

流量控制：接收方 -> 发送方 慢点（端到端）， 流量控制只让发送方的发送速率不要太快，流量控制是使用滑动窗口来实现的。
拥塞控制：拥塞控制考虑整个网络，是全局性的考虑。可以通过慢启动算法，拥塞窗口实现

四、计算机组成原理

机组基本上就没学过，哎

1、冯诺依曼结构的特点

  1. 计算机由运算器，控制器，存储器，输入输出设备五部分组成
  2. 指令和数据以同等地位存储在存储器中，按地址访问
  3. 数据均用二进制表示

2、编译和解释的区别

都是翻译的方式
编译：将源代码转化成目标文件，然后执行
解释：边翻译边执行，无目标文件

3、机器字长、指令字长、存储字长区别

机器字长：计算机可直接处理的二进制数据位数
指令字长：一个指令字包含的二进制代码位数
存储字长：一个存储单元存储的二进制代码长度

4、CPI，MIPS

CPI：执行一条指令所需时钟周期数
MIPS：每秒多少百万条指令=主频/CPI

5、计算机中为什么采用二进制表示数据

1、实现简单，可以电子元件高电位表示1，低电位表示0这样

2、二进制运算法则少，运算简单，有利于简化计算机硬件结构

6、CISC和RISC的区别

CISC复杂指令集，RISC是精简指令集。

7、cache的作用

连接cpu和内存

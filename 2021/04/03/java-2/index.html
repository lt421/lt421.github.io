  2. int[] arr = {1,2,3}

二、面向对象（基础）

谈谈自己理解 （借鉴大话设计模式）

什么是对象：万物皆对象，对象是一个自包含（组件不依赖其他组件,能够以独立的方式供外部使用）的实体，用一组行为（方法）和特性（属性）来标识

什么是类：具有相同属性和功能的对象的集合

面向对象的好处：

2.1、对象在内存中存在形式

2.2、类和对象的内存分配机制

2.3、方法调用机制

2.4、重载

允许方法名相同

细节：

  1. 不能仅通过返回类型区别两个重载方法

2.5、可变参数

细节：

  1. 多个参数时，可变参数放最后
  2. 一个形参列表中只能有一个可变参数
  3. 本质是数组（参数可以为数组）

2.6、作用域

全局变量：属性（成员变量）

局部变量：方法或者代码块定义的变量

2.7、构造方法/构造器

系统默认生成无参构造器（一旦定义构造器则不自动生成）

2.8、对象创建流程分析（面试）

  1. 加载Person类信息（方法区）
  2. 堆开辟空间
  3. 对象初始化
    默认初始化：age = 0 ; name = null
    显式初始化：age = 90; name = null
    构造器初始化：age = 20 ;name = ”小倩“（常量池）

2.8、this关键字

简单说，哪个对象调用就是哪个对象引用

细节：

  • 构造器内使用this,只能调用其他构造器且必须放在第一行

3.2、封装

封装：把抽象出的数据和对数据操作的方法封装在一起，数据被保护在内部，程序的其他部分只能通过方法来对数据进行操作

好处：

  1. 隐藏细节
  2. 对数据进行验证，保证安全，合理

实现封装：属性定义为private ，提供get set 方法

3.3、继承

好处：

  1. 提高代码复用性
  2. 提高拓展性和维护性

细节：

1、子类继承了所有属性和方法，私有属性和方法不能直接访问，使用supper

2、子类必须调用父类的构造器，完成父类的初始化

3、创建子类对象时，不论使用子类的哪个构造器，都会调用父类的构造器，若父类没有默认无参构造器需使用super指定使用父类哪个构造器，负责不会通过编译。

4、super()使用必须放在构造器内第一句（不能和this()同时使用）

5、Object是所有类的基类（idea框架键 ctrl + h 可以看到类的基础关系）

6、父类构造器的调用将一直追溯到Object类

7、java单继承（多个？ – 接口/抽象类）

8、不能滥用基础，需要满足“is - a”关系

3.3.1、继承本质详解

3.3.2、supper

作用：

  1. 访问父类方法获变量（当方法或变量重名）
  2. 分工明确，父类变量由父类初始化，子类变量自己初始化

3.3.3、方法重写/覆盖（override）

细节：

  1. 除返回类型（方法名，参数列表）都必须一样，返回类型可以为父类返回类型的子类
  2. 子类不能缩小父类的访问限制

3.4、多态

方法或对象拥有多种状态

1、方法多态：方法的重写和重载

2、对象多态

  1. 一个对象的编译类型和运行类型可以不一样
  2. 编译类型在定义对象时就确定了，不能改变
  3. 运行类型是可以变换的
  4. 编译类型在 = 左边 ；运行类型在 = 右边

3.4.1、细节

1、向上转型：父类的引用指向子类的对象

  • 可以调用父类中所有成员
  • 不能调用子类特有成员
  • 最终运行结果看子类的具体实现

2、向下转型：子类的引用指向父类的对象

  • 只能强转父类的引用，不能强转父类的对象

  • 父类的引用必须指向当前目标类型的对象

  • 可以调用子类中所有成员

    ++

instanceof 判断的是运行类型

3.4.2、动态绑定机制（非常非常重要）

  1. 调用对象方法时，会和运行类型动态绑定
  2. 调用方法属性时，没有动态绑定，哪里声明哪里使用

3.4.3、多态应用

多态数组：数组定义为父类型，存放子类对象

多态参数：形参类型为父类类型，允许实参为子类类型

3.5、Object类

3.5.1、equals 与 ==

== 基本类型判断值是否相等，引用类型判断地址是否相等（引用的是一个对象）

equals 只能判断引用类型，但是往往重写此方法判断是否相等

3.5.2、finalize

几乎不用，自动释放内存，应付面试

3.5.3、hashCode

  1. 提高具有哈希结构的容器的效率！
  2. 两个引用，如果指向的是同一个对象，则哈希值肯定是一样的！
  3. 两个引用，如果指向的是不同对象，则哈希值是不一样的
  4. 哈希值主要根据地址号来的！， 不能完全将哈希值等价于地址。
  5. 在集合时，重写 hashCode()

3.5.4、toString

默认返回：全类名+@+哈希值的十六进制，【查看 Object 的 toString 方法】

子类往往重写 toString 方法，用于返回对象的属性信息

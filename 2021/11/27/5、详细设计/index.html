二、程序复杂度的定量度量

2.1、McCabe方法

就是将流程图转为数据流图，注意复合条件的转换即可

2.2、Halstead方法

根据程序中和操作数和运算符的总数来定量程序的复杂度

N1 为程序中运算符出现的总次数 ，N2为程序中操作数出现的总次数，实际长度N

N = N1 + N2

若详细设计结果中使用不同的运算符(操作数的种数)个数为n1，不同操作数（变量和常数）的个数（运算符的种数）为n2, 预测长度H

H = n1*log2(n1) + n2 * log2(n2)

例：

z = 0;

while X > 0

​ Z = Z + Y;

​ X = X - 1;

end_while;

print(Z);

通常把常数和变量是为操作数，其他符号（逗号、分号、圆括号、方括号、算数运算符、逻辑运算符）都视为运算符

把所有总数总是成对出现或以三个一组方式出现的符号视为一个运算符

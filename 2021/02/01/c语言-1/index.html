c语言简介

基本数据类型

一、C语言发展

与UNIX 密不可分，汇编语言编写的UNIX，UNIX使用c语言改写，从此编写操作系统的主要语言

至今仍是排名靠前语言

主流版本： c89(标准c)，c99

c语言特点：

  • 代码级别的跨平台
  • 允许直接访问物理地址，对硬件进行操作：（操作系统，数据库，杀毒软件，C语言，驱动…）
  • 传值方式：按值/按指针
  • 没有对象 - 结构体

二、C程序运行机制

  1. 编辑：编写.c文件，就是源代码
  2. 编译：将.c文件翻译成目标文件(.obj) //计算机底层
  3. 链接：将目标文件.obj + 库文件 生成可执行文件(.exe) //计算机底层
  4. 运行：执行.exe文件，得到运行结果

什么是编译：

​ 有了c源文件，通过编译器将其编译成obj文件(目标文件)

​ 如果程序没有错误，没有任何提示，但在Debug目录下会出现一个（程序名）.obj文件，该文件称为目标文件

什么是链接：

​ 有了目标文件（.obj文件）,通过链接程序将其和运行所需要的C库文件链接成exe文件（可执行文件）

​ 如果程序没有错误，没有任何提示，但在Debug目录下会出现一个项目名.exe文件，该文件称为可执行文件

​ 需要链接库文件？ 使用printf()等函数是c程序库中提供，并不是自己写的

​ 链接后的.exe比.obj大很多（还包含了库文件）

什么是运行：

​ 链接后生成的.exe文件（二进制文件）

​ 可以在控制台直接运行

四、变量

变量：变量相当于内存中一个存储空间的表示

输出格式：%d ,%f,%c,%s %ld,%lld

4.1、变量的数据类型

操作系统，系统位数，编译器都会导致部分数据类型字节长度不一样，使用sizeof()得到的字节长度。

实际工作中，c程序通常在linux/unix中运行

4.1.1、整型

表示整数的数据类型

数据类型 存储大小(字节) 范围(最高位表示符号位)
char / signed char 1 -128 到 127
unsigned char 1 0 到 255
int / signed int 2 或 4 -2^15 到 2^15 - 1 或 -2^31 到 2^31 - 1
unsigned int 2 或 4 0 到 65535（2^15 -1） 或 0 到 2^31 -1
short / signed short 2 -2^15 到 2^15 - 1
unsigned short 2 0 到 65535（2^15 -1）
long / signed long 4 -2^31 到 2^31 - 1
unsigned long 4 0 到 2^31 -1

4.1.2、浮点型

表示小数的数据类型

  • 浮点数是近似值
  • 尾数部分可能丢失，造成精度损失
  • 通常使用double(默认为double)
  • printf 默认保留后六位
数据类型 存储大小(字节) 范围（符号位+指数位+尾数为） 精度
float 单精度 4 1.2E-38 到 3.4E+38 6位小数
double 双精度 8 2.3E-308 到 1.7E+308 15位小数

char 表示字母或数字 ‘A’ , ‘1’

char的底层是整数:

  • 读取反之

ASSIC:

4.1.4、布尔型

c89没有定义布尔类型

表达不直观，使用宏定义解决如

c99提供_Bool型,_Bool仍然是整数型，但只能赋值为0或1，非零的值会被存储为1

4.2、基本数据类型转换

4.2.1、自动类型转换

  • 多种类型数据混合运算时，系统将都转为精度最大的类型

  • 赋值运算中可能发送降低精度，如 float -> double

4.2.2、强制类型转换

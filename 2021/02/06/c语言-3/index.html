三、函数

  • 完成某一功能的程序指令（语句）的集合

3.1、头文件

在一个文件中调用另一个文件中的函数 => 头文件

  • 拓展名为.h的文件，包含了c函数的定义和声明，被多个源文件引用共享
  • #include 文件包含命令 ，用来引入头文件 ，#include 属于预处理指令
  • #include 处理的过程就是将头文件的内容插入到该命令所在的位置，从而把头文件和源文件连接成一个文件（等同于复制粘贴）
  • 建议把所有的常量，宏，系统全局变量和函数原型写在头文件中，需要是引用

Notes:

  1. 引用头文件相当于复制
  2. 源文件的名字可以和头文件不同
  3. <> 引用系统头文件 （也可以引用自定义头文件）
    “ ” 引用自定义头文件
  4. 引用多次和引用一次作用一样，代码层面有防止重复引入机制
  5. 头文件中只能包含变量和函数的声明，不能包含定义

四、变量的作用域

  1. 函数内部声明/定义的局部变量，作用域仅限函数内部
  2. 代码块声明/定义的局部变量，作用域代码块
  3. 局部变量和全局变量重名，优先使用局部变量
  4. 函数外部定义即为全局变量（通常放在头文件中）

初始化问题：

  • 局部变量需要初始化后使用，否则可能异常退出

  • 全局变量系统自动初始化

    数据类型 默认化初始值
    int 0
    char ‘\0’
    float 0.0
    double 0.0
    pointer 指针 null

Notes

  1. 全集变量（Global Variable）保存在内存的全局存储区中，占用静态的存储单元，它的作用域是整个程序，也就是所有的代码文件
  2. 局部变量（Local Variable）保存在栈中，函数被调用是才分配存储单元
  3. C语言规定，只能从小的作用域向大额作用去寻找变量，而不能反过来，使用更小的作用域的变量

4.1、static关键字

static关键字在c语言中比较常用，使用恰当能够大大提高程序的模块化特性，有利于拓展和维护

局部变量使用static修饰：

  1. 称为静态局部变量
  2. 自动初始化
  3. 存储于内存的静态存储区，只会初始一次，即使函数返回，它的值也保持不变

全局变量使用static修饰：

  1. 普通全局变量对整个工程可见，其他文件可以使用extern外部声明后直接使用，也就是其他文件不能再定义一个相同名字的变量了（否则编译器会认为他们是一个变量），静态全局变量仅对当前文件可见，其他文件不可访问。
    报错：
    CMakeFiles\day01.dir/objects.a(static.c.obj):G:/clion_project/basic/day01/static.c:9: multiple definition of `n’
    CMakeFiles\day01.dir/objects.a(function.c.obj):G:/clion_project/basic/day01/head/function.c:39: first defined here
    若要使用：
    extern int n;
  2. 定义不需要与其他文件共享的全局变量时，加上static关键字能够有效的降低程序之间的耦合，避免不同变量的冲突，且不会误用

函数使用static:

  1. 非静态函数可以在另一个文件中通过extern引用
  2. 静态函数只能在声明它的文件中可见，其他文件不能引用该函数

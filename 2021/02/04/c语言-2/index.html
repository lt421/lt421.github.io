运算符与指针入门（再一次学习c语言果然和初学不一样哈哈）

一、指针入门

1.1、指针应用案例

  1. 写一个程序，获取一个int变量num的地址，并显示到终端
  2. 将num的地址赋值给指针ptr，并通过ptr去修改num的值
  3. 画出案例的内存布局图

1.2、指针细节

  • 基本数据类型，都有对应的指针类型
  • 此外还是指向数组的指针，指向结构体的指针，执行共用体的指针，（二级指针，多级指针）后续学习

1.3、值传递和地址传递

1、值传递

将变量指向的存储内容，在传值/赋值时，拷贝一份给接收变量

2、地址传递

如果是指针或者数组在传值/赋值时，指针传递存储地址，数组传递首地址

分别分析以下两种情况

二、 常量

  1. 固定值，不能修改
  2. 可以是任何基本数据类型
  • 整数常量
  • 浮点常量
  • 字符常量
  • 字符串常量

2.1、常量的定义

一般使用大写+下划线定义常量

  • 使用#define 预处理器

  • 使用const关键字

区别：

  1. const带类型，define不带类型

  2. const在编译，运行时才起作用，define在编译的预处理阶段起作用

  3. define只是简单的替换，没有类型检查，简单的字符串替换会导致边界效应

  4. const可以进行调试，而define不行，预编译阶段以及替换，调试的时候就没有了

  5. const不能重复定义，而define可以通过undef取消某个符号的定义，再重新定义

  6. define还可以配合#ifdef,#ifndef,#endif 来使用，使代码更加灵活。比如通过#define来启动或关闭调试信息

三、运算符

位运算符：

&：按位与

|：按位或

^: 按位异或

~: 按位取反

3.1、算数运算符

i++ 与 ++i

3.2、关系运算符

false:0

true:非零，默认1

3.3、逻辑运算符

短路现象：

&& 第一个条件为false，后面的条件不再判断 （可以用来避免空指针异常）

|| 第一个条件为true，后面的条件不再判断

3.4、赋值运算符

无需补充

3.5、位运算符

1、进制转换

1.1、其他进制转十进制

1011 = 1 * 2^0 + 1 * 2^1 + 0 * 2^2 + 1 * 2^3 = 1 + 2 + 8 = 11 (我的做法：记住这个：256 128 64 32 16 8 4 2 1，然后对应二进制位，1的话就加，0就不加，得到结果就是十进制)

0123 = 3 * 8^0 + 2 * 8^1 + 1 * 8^2 + 0 * 8^3 = 83

0X34A = A(10) * 16^0 + 4 * 16^1 + 3 * 16^2 = 10 + 64 + 768 = 842

1.2、十进制转其他进制

十进制转二进制：模2取余

八进制转二进制：模8取余

十六进制转二进制：模16取余

1.3、二进制转其他进制

二进制转八进制：三位一组转为十进制再拼接

二进制转十六进制：四位一组转十进制再拼接

1.4、其他进制转二进制

八进制转二进制：每一位转为对应的三位的二进制数

0237 => (000 0)10 011 111

十六进制转二进制：每一位转为对应的四位的二进制数

0x23B => (00)10 0011 1011

2、位运算

原码，反码，补码：

  1. 二进制的最高位是符号位，0表示正数，1表示负数
  2. 正数的原码，反码，补码都一样
  3. 负数：
    反码 = 原码取反（符号位不变）
    补码 = 反码 + 1
  4. 0的反码，补码都是0
  5. 计算机是按照反码运算的
运算符 描述
& 按位与 (1,x -> 1)
| 按位或 (0,x -> 0)
^ 按位异或（1,0 -> 1）
~ 按位取反
<< 左移 （左边丢弃，右边补1）
>> 右移 （正数左补0，负数左补1，右边丢弃）

3.6、三元运算符

细节：表达式1和表法式2要为可以赋给接收变量的类型（或者可以自动转换），否则可能会有精度损失

四、标识符

变量命名：数字，字母，_和$，数字不能打头

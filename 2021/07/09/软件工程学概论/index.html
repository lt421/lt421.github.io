1、软件工程学概论

发表于2021-07-09|更新于2022-03-18|软件工程
|字数总计:944|阅读时长:2分钟|阅读量:|评论数:

考研专业课，冲冲冲

  1. 软件
  2. 软件工程
  3. 软件生命周期
  4. 软件过程

考纲：掌握软件工程基本概念。包括软件危机、软件工程、软件生命周期、软件过程。

一、软件

程序 + 数据 + 文档

二、软件工程

1、软件发展

  • 程序设计
  • 程序系统 （出现软件危机 -> 软件工程）
  • 软件工程
  • 第四阶段

2、软件危机

定义：计算机软件开发和维护中遇到的一系列严重的问题

表现：

  1. 开发成本和进度估计不准
  2. 用户对已交付的软件不满意
  3. 软件产品的质量靠不住
  4. 软件的可维护性查
  5. 软件没有适当的文档资料
  6. ….

3、软件工程

定义：软件工程是指导计算机软件开发和维护的一门工程学科，采用工程的概念，方法，技术来开发和维护软件，把经过时间考量而正确的管理技术和当前能够得到的最好技术方法结合起来，以经济的开发出高质量的软件并维护

三、软件生命周期

定义：软件产品定义、设计、开发、维护、废弃的过程

四、软件过程

软件过程：获得高质量软件所需要完成一系列任务的框架，规定了完成各项任务的工作步骤

(who when what how )

1、瀑布模型

最早，最广泛应用

  1. 阶段间具有顺序性和依赖性
    顺序性：前一步完成才能进行后一步；
    依赖性：前一阶段的输出文档是后一阶段的输入文档
  2. 推迟实现
    忽视设计的重要性，直接开始码代码；（磨刀不误砍柴工）
  3. 质量保证
    每个阶段结束都需要文档；
    对文档进行评审

问题：过于理想化，不可能不出错或者遇到需求的变更

2、快速原型模型

根据主要需求，快速建立原型，反复和用户交流，根据需求再进行开发

优点：确定需求上优于瀑布模型；

原型可以用于指导开发或者作为系统的一部分；

3、增量模型

分批的向用户提交产品

如：开发喜瑞斯的时候先完成后台开发，可以先进行数据的添加；

直接划分若干构件，并行执行，风险更大的增量模型

4、螺旋模型

风险：比如未能按规定时间完成；做到一半没钱了；程序员删库跑路或跳槽；对手已经做出类似软件等等

加入风险分析，适用于大型软件的开发；

可理解为：在每个阶段之前都增加了风险分析过程的快速原型模型

优点：大型软件项目很好的风险控制

缺点：需要有风险评估的经验；普及不如前面

5、喷泉模型

面向对象方法的生命周期模型，迭代、无缝

优点：同步开发、提升开发效率，节省开发时间，适用于面向对象软件

缺点：可能随时加各种信息，需求，资料，文档管理和审核困难

6、Rational 统一过程

完整且完美软件过程，二维

7、微软过程

微软公司独特的软件开发过程；

综合了rational过程和敏捷开发的优点；

8、敏捷过程与极限编程

敏捷软件开发宣言：

  1. 个体和交互胜过过程和工具

  2. 可以工作的软件胜过面面俱到的文档

  3. 客户合作胜过合同谈判

  4. 响应变化胜过遵循计划

极限编程（extreme programming） - 敏捷开发最负盛名

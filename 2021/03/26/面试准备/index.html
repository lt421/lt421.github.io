面试前一天准备下人生第一次面试

面试准备

一、springcloud

1、什么是微服务

架构风格、把一个项目拆分成多个服务，多个服务独立运行

2、为什么要使用微服务

传统j2ee缺陷：拓展性查，维护成本高，可靠性低

所有模块都堆砌在一起，耦合度高，维护困难；拆分每个模块代码量少，易于维护

存储在一个数据库，存储方式单一；微服务不同的服务可以存储不同的数据库，比如redis或mysql

每个模块可以使用不同的技术实现；

3、springcloud组件

  1. 服务发现 - Nacos 阿里巴巴 注册中心 替代原生的 Eureka

  2. 服务调用 - feign 一个微服务调用另一个微服务的接口 整和了ribbon

  3. 熔断器 - Hystrix
    分布式 - 将单独的服务部署在不同的服务器上
    提供延迟和容错的功能，在一个服务出现问题的时候，没有熔断机制会导致系统挂掉

  4. 配置中心 - nacos - 替代原生的springcloud config

    需要变更的参数、变量等从代码中分离出来独立管理，以独立的配置文件的形式存在

  5. springcloud gateway 替代zull
    API 网关是介于客户端和服务器端之间的中间层，所有的外部请求都会先经过 API 网关这一层
    作用：
    1）请求转发
    2）负载均衡 轮询/权重/最少请求时间
    3）跨域其他配置（拦截器/全局异常处理）

二、springboot

1、什么是springboot

框架，我理解：管家管理spring springMVC mybaties 框架，约定大于配置，很少的配置快速进行开发

2、日志

springboot 使用 slfj + logback

3、restful

URI:/资源名称/资源标识

HTTP请求方式区分对资源crud操作

c post

r get

u put

d delete

三、springSecurity (Shiro)

权限管理

认证：用户身份是否合法

授权：访问资源是否有权限

授权使用 RBAC

R: role/resource

B:base

AC:access control

基于session认证：

session_id 存放在cookie中，这样用客户端请求时带上 session_id 就可以验证服务器端是否存在
session 数据，以此完成用户的合法校验

分布式系统认证 token

Oauth2.0 第三方用户可以使用用户存储在其他服务器上的资源，而不需要用户名和密码，用户进行授权

  1. 授权码模式 用户拿着授权码请求token
  2. 简化模式 直接发放token

  3. 密码模式 直接拿到用户名和密码

  4. 客户端模式 直接跳转

token ： 字符串 包含了用户信息

jwt :生成token的方式

四、Redis

数据类型： 数值 字符串 哈希 集合 有序集合 列表

1、缓存穿透

缓存中查不到，在数据库也查不到

解决：这时也可以将key-value对写为key-null

2、缓存击穿

缓存击穿是指缓存中没有但数据库中有的数据（一般是缓存时间到期），这时由于并发用户特别多，同时读缓存没读到数据，又同时去数据库去取数据，引起数据库压力瞬间增大，造成过大压力

解决：热点数据，永不过期

互斥锁

3、缓存雪崩

数据大批量到过期时间，而查询数据量巨大，引起数据库压力过大甚至down机

缓存击穿指并发查同一条数据，缓存雪崩是不同数据都过期了

解决：缓存数据的过期时间设置随机，防止同一时间大量数据过期现象发生

热点数据不过期

五、Mysql

索引：增加查询效率，但会降低其他操作的效率（每次更新索引）

b树或者b+树，多叉树：重新组织节点，降低树的高度，提高读写效率

ACDI

原子性、一致性、隔离性、持久性

事务隔离级别：

  1. 脏读 （dirty read）
  2. 读/写提交 (read commit)
  3. 可重复读 (repeated read)
  4. 序列化 (serializable)

六、svn/git/idea

项目开发使用，较为熟悉
